PocaskE&M

실시간 광고 플랫폼 안정성 확보 및 인프라 구축

도전 과제

  • DB 데드락: 일일 광고 송출량 5배 폭증으로 인한 DB 병목 및 데이터 유실(일 100~200건)
  • 인프라 한계: 클라우드 기반 크롤링 시스템의 잦은 IP 차단 및 높은 운영 비용

나의 역할 및 해결 방안

  • 이벤트 큐 도입: Redis를 활용해 DB 쓰기 작업을 비동기 벌크 처리 방식으로 전환
  • 온프레미스 인프라 설계: 4대의 고가용성 온프레미스 서버로 인프라를 직접 구축하여 처리량 극대화

핵심 성과

  • 피크 타임 DB CPU 사용률 90%대에서 안정적인 수준으로 감소
  • 데이터 정합성 100% 확보 및 서비스 전환율 상승 기여
  • IP 차단 실패율 감소 및 장애 복구 시간 단축 등 운영 효율성 개선

지식 공유 문화 조성

Kafka 운영 경험을 바탕으로 팀 내 기술 세미나를 주도하여 팀원들의 기술 이해도를 높이고, 팀 전체의 기술 역량 상향 평준화에 기여했습니다.

그로잉랩

금융 데이터 플랫폼 성능 최적화 및 아키텍처 개선

도전 과제

  • 성능 병목: 대용량 JSON 파일 로드 구조로 인한 3초 이상의 느린 API 응답 속도
  • 유지보수성 저하: Express, Hapi로 파편화된 레거시 백엔드 구조로 인한 높은 개발 비용

나의 역할 및 해결 방안

  • 아키텍처 전환 개발 참여: 테크리더의 방향에 따라 NestJS + TypeORM 기반 단일 프로젝트로 통합 리팩토링의 핵심 개발을 담당
  • 데이터베이스 설계 최적화: 모놀리식 JSON 구조를 RDB 기반 도메인별 API로 분리 설계

핵심 성과

  • API 응답 속도 96% 개선 (3초 → 100ms)으로 사용자 경험 최적화
  • 신규 기능 개발 시간 50% 단축 등 팀 전체 생산성 향상

Concent

소셜 미디어 앱 인프라 비용 최적화

도전 과제

스타트업에게 부담되는 월 60만원의 AWS S3 스토리지 비용.

나의 역할 및 해결 방안

HLS 스트리밍 구조를 도입, 원본 영상 대신 분할된 .ts 세그먼트 파일을 업로드하도록 아키텍처를 전환하여 스토리지 사용량과 트래픽 비용을 최적화했습니다.

핵심 성과

  • 월간 AWS S3 비용 76% 절감 (60만원 → 14만원)
  • CodePush 기반 핫픽스 파이프라인 구축으로 배포 리드타임 단축

브릭메이트

대규모 커머스 플랫폼 고도화

주요 경험 및 역량

  • 대기업의 표준 개발 프로세스 및 스크럼 방식 체득
  • 명확하게 구분된 역할과 책임을 기반으로 한 협업 방식 학습
  • 레거시 시스템 분석 및 안정적인 유지보수 역량 강화
  • 대규모 트래픽 환경에서의 장애 대응 및 이슈 관리 경험

신입 개발자 시기에 경험한 이 프로젝트는 기술적 성장보다는, 대규모 조직이 어떻게 협업하고 시스템을 안정적으로 운영하는지에 대한 귀중한 인사이트를 얻는 기회였습니다. 이때 배운 체계적인 협업 방식과 책임감은 이후 스타트업 환경에서 주도적으로 프로젝트를 이끌어 나가는 데 튼튼한 기반이 되었습니다.
