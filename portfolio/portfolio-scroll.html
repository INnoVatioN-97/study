PocaskE&M

실시간 광고 플랫폼 안정성 확보 및 크롤링 인프라 최적화

도전 과제

  • DB 데드락: 일일 광고 송출량 5배 폭증으로 인한 DB 병목 및 데이터 유실(일 100~200건)
  • 인프라 비효율/한계: 클라우드 크롤링 인스턴스 43대의 운영 비용 절감 및 잦은 IP 차단 이슈
  • 데이터 유실: 재배포 시 Kafka 로컬 큐 작업 유실 및 메시징 구조 불안정성

나의 역할 및 해결 방안

  • 이벤트 큐 도입: Redis를 활용해 DB 쓰기 작업을 비동기 벌크 처리 방식으로 전환하여 Deadlock 해결
  • 인프라 이원화: 클라우드 인스턴스를 29개로 34% 감축하고, 고성능 온프레미스 서버를 병행 구축하여 처리량 유지 및 고가용성 확보
  • Kafka 메시징 리팩토링: Emit 방식의 메시지 전송을 제거하고 Send 기반으로 전환, 처리 완료 후 Commit하는 구조로 변경하여 데이터 유실 문제 근본적 해결

핵심 성과

  • 피크 타임 DB CPU 사용률 90%대에서 안정적인 수준으로 감소
  • 데이터 정합성 100% 확보 및 서비스 전환율 상승 기여
  • 클라우드 인프라 34% 감축(43개 → 29개)을 통해 운영 효율성 및 비용 최적화 달성
  • 메시징 구조 개선을 통해 재시작 시에도 이어서 처리 가능, 장애 복구 속도 단축 및 안정성 강화

지식 공유 및 기술 개선

Kafka 운영 경험을 바탕으로 팀 내 기술 세미나를 주도하여 팀원들의 기술 이해도를 높이고, 팀 전체의 기술 역량 상향 평준화에 기여했습니다.

최근에는 세미나에서 논의되었던 기술 부채를 해결했습니다.
기존 emit 방식의 메시지 유실 가능성을 보완하기 위해 send 방식으로 전환하여 데이터 신뢰성을 높였습니다.
이 개선을 통해 Consumer Lag 모니터링이 가능해져, 트래픽 피크 타임의 크롤링 지연 상태를 명확히 파악하고 대응할 수 있게 되었습니다.

그로잉랩

금융 데이터 플랫폼 성능 최적화 및 아키텍처 개선

도전 과제

  • 성능 병목: 대용량 JSON 파일 로드 구조로 인한 3초 이상의 느린 API 응답 속도
  • 유지보수성 저하: Express, Hapi로 파편화된 레거시 백엔드 구조로 인한 높은 개발 비용

나의 역할 및 해결 방안

  • 아키텍처 전환 개발 참여: 테크리더의 방향에 따라 NestJS + TypeORM 기반 단일 프로젝트로 통합 리팩토링의 핵심 개발을 담당
  • 데이터베이스 설계 최적화: 모놀리식 JSON 구조를 RDB 기반 도메인별 API로 분리 설계

핵심 성과

  • API 응답 속도 96% 개선 (3초 → 100ms)으로 사용자 경험 최적화
  • 신규 기능 개발 시간 50% 단축 등 팀 전체 생산성 향상

Concent

소셜 미디어 앱 인프라 비용 최적화

도전 과제

핵심 성과

  • 월간 AWS S3 비용 76% 절감 (60만원 → 14만원)
  • CodePush 기반 핫픽스 파이프라인 구축으로 배포 리드타임 단축
