PocaskE&M

대규모 광고 솔루션 백엔드 개발 및 신규 리워드앱 구축

핵심 프로젝트 및 모듈 개발

1. 광고 리워드앱 P*** (진행 중)

Spring Boot 기반 신규 서비스 구축 주도

  • Spring Boot 기반 REST API, MyBatis + MySQL 스키마 설계 및 개발
  • JWT 듀얼 토큰 인증 체계 및 Redis 기반 포인트 도메인(지갑, 적립, 교환) 모듈 개발
  • 모바일 바우처 플랫폼(Woori Card, 슈퍼콘 등) 연동 API 개발

주요 성과: Swagger 스키마 표준화를 통한 FE 및 외부 파트너 협업 속도/품질 향상

2. Kafka 기반 크롤링 인프라 고도화

클라우드/온프레미스 혼합 아키텍처 설계

  • 크롤링 인스턴스 43개에서 29개로 32% 감축 및 PM2 클러스터링 적용 (처리량 유지)
  • 고가용성 확보를 위한 온프레미스 고성능 크롤러 인프라 이원화 구성 주도
  • 메시지 유실 문제 해결을 위한 Emit 기반 Producer/Consumer 구조를 Send 기반으로 리팩토링

주요 성과: 클라우드 인프라 감축을 통한 운영 비용 절감 및 데이터 신뢰성 확보

3. 광고 솔루션 '포인트 크러쉬' 안정화

대규모 트래픽 DB 데드락 문제 해결

  • 일일 광고 송출량 5배(10만→50만 건) 급증에 따른 DB 데드락 문제 발생
  • Redis 이벤트 큐 도입, 부가 프로세스를 비동기 벌크 처리로 전환하여 해결
  • Promise.all 구조 개선 및 Chunk 단위 병렬 처리/인덱싱 최적화 적용

주요 성과: 피크 타임 DB CPU 사용률 90%대에서 안정화 및 데이터 유실 문제 해결

4. 실시간 통계 알림 서버 구축

데이터 기반 의사결정 체계 마련

  • Spring Boot 기반 배치 서버를 통해 광고 소비량 및 시스템 이슈 실시간 감지
  • 실시간 통계 데이터 파이프라인 설계 및 구축

주요 성과: 장애 대응 시간 단축 및 데이터 정합성 확보, 비즈니스 전략 수립을 위한 기반 마련

기술 리더십 및 협업 문화

지식 공유 및 기술 개선

Kafka 운영 경험을 바탕으로 팀 내 기술 세미나를 주도하여 팀원들의 기술 이해도를 높이고, 팀 전체의 기술 역량 상향 평준화에 기여했습니다.

기존 emit 방식의 메시지 유실 가능성을 보완하기 위해 send 방식으로 전환하여 데이터 신뢰성을 높였습니다.
이 개선을 통해 Consumer Lag 모니터링이 가능해져, 트래픽 피크 타임의 크롤링 지연 상태를 명확히 파악하고 대응할 수 있게 되었습니다.

업무 자동화 및 개발 환경 개선

주간 업무 보고 자동화

n8n 워크플로우 자동화 툴을 도입하여 주간 업무 보고 작성을 자동화했습니다. 분산된 업무 데이터를 자동으로 취합하고 리포트 형태로 가공하여, 단순 반복적인 행정 업무 시간을 단축하고 팀이 개발 본연의 가치에 집중할 수 있도록 기여했습니다.

AI 기반 Hotfix 워크플로우 (개발 중)

프로덕션 에러 발생 시 AI 에이전트가 로그를 분석하고 즉시 핫픽스 PR을 자동으로 생성하는 워크플로우를 개발 중입니다. 이를 통해 장애 인지부터 복구까지의 시간을 획기적으로 단축하고, 더욱 견고하고 신속한 대응이 가능한 운영 환경을 구축하고자 노력하고 있습니다.

주요 성과: 개발 표준화를 통해 신규 기능 개발 시간 50% 단축 및 팀 생산성 향상

2. 데이터 처리 속도 96% 최적화

파일 기반 데이터 구조를 RDB로 재설계

  • 대용량 JSON 파일을 로드하는 구조를 RDB 기반 도메인별 정규화된 테이블로 마이그레이션
  • 인덱싱 및 쿼리 최적화를 통해 데이터 조회 성능을 극적으로 개선

Concent

소셜 미디어 앱 인프라 비용 최적화

핵심 프로젝트 및 성과

1. 영상 스트리밍 인프라 비용 최적화

HLS 아키텍처 도입 및 스토리지 효율화

  • 통영상 업로드 방식을 FFmpeg 기반의 HLS(HTTP Live Streaming) 분할 업로드 구조로 전환
  • S3 스토리지 수명 주기 정책 및 CloudFront 캐싱 전략 최적화

주요 성과: 월간 AWS S3 비용을 60만원에서 14만원으로 76% 절감

2. 핫픽스 배포 파이프라인 구축

CodePush 도입으로 대응 속도 향상

  • 스토어 심사 없이 즉시 배포 가능한 CodePush 기반 OTA 업데이트 시스템 구축
  • GitHub Actions 기반의 자동화된 빌드 및 배포 파이프라인(CI/CD) 설계

주요 성과: 긴급 버그 수정 배포 리드타임을 수일에서 수분 단위로 단축

인사이트 및 성장

신입 개발자 시기에 경험한 롯데홈쇼핑 프로젝트는 기술적 성장뿐만 아니라, 대규모 조직이 어떻게 협업하고 시스템을 안정적으로 운영하는지에 대한 귀중한 인사이트를 얻는 기회였습니다. 이때 배운 체계적인 협업 방식과 책임감, 그리고 1인 개발 프로젝트를 통해 얻은 주도적인 문제 해결 능력은 이후 스타트업 환경에서 팀을 이끌고 프로젝트를 성공시키는 튼튼한 기반이 되었습니다.
